# SuperPmAgent Business Skills

| Skill | Path | Use When |
|---|---|---|
| add-db-field | `add-db-field/SKILL.md` | A requirement adds or exposes a persistent model field. |
| add-api-endpoint | `add-api-endpoint/SKILL.md` | A requirement needs a new backend route or response shape. |
| add-ui-form | `add-ui-form/SKILL.md` | A requirement adds user input to an existing UI flow. |
| add-list-page | `add-list-page/SKILL.md` | A requirement adds a new list view, tab, or filtered collection display. |
| add-detail-page | `add-detail-page/SKILL.md` | A requirement adds a detail section, info display, or computed field view. |
| add-filter | `add-filter/SKILL.md` | A requirement adds filtering, sorting, or search to an existing list. |
| gen-feishu-design | `gen-feishu-design/SKILL.md` | PM writes requirements in a Feishu doc — auto-generate architecture design doc, task assignment doc, and Bitable task tracker. |

Distilled business patterns should be added as new skill folders after review.
